Approche 1 : TF-IDF (Baseline)

III. Pipeline NLP

Comment fonctionne TF-IDF ?

TF-IDF = Term Frequency × Inverse Document Frequency

  • TF : Fréquence du mot dans le document
  • IDF : Rareté du mot dans le corpus
  • Résultat : Vecteur sparse (creux)
"Python developer" →
[0, 0, 0.8, 0, 0.6, 0, 0, ...]
# 7000 dimensions, mostly zeros

Pipeline de prétraitement

CV Brut → Nettoyage
→ Tokenization → Stopwords
→ Lemmatization → TF-IDF

Limite : Ne capture pas le sens sémantique
"développeur" ≠ "programmer" pour TF-IDF

Approche 2 : Sentence Transformers (Embeddings)

III. Pipeline NLP

Comment fonctionnent les Embeddings ?

Un réseau de neurones pré-entraîné transforme le texte en vecteur dense.

  • Sentence Transformers : Modèle BERT optimisé
  • Capture le sens sémantique
  • Vecteur dense : 384 dimensions
  • Multilingue : 50+ langues
"Python developer" →
[0.23, -0.45, 0.12, ...]
"Développeur Python" →
[0.22, -0.44, 0.13, ...]
# Vecteurs similaires = même sens!

Processus d'embedding

Texte CV → Tokenization
→ BERT Encoder → Pooling
→ Vecteur 384D

Modèle : paraphrase-multilingual-MiniLM-L12-v2

Entraîné sur 1 milliard+ de paires de phrases

Support de 50+ langues
